Chapter 6 Bayesian Zero-Inflated Count Models in Stan

In the previous post, we covered logistic regression for binary outcomes and negative binomial regression for overdispersed count data. Both are workhorses of applied statistics, but they share a quiet vulnerability: neither handles an excess of zeros particularly well. If you’ve ever modeled the number of doctor visits in a population, or species counts across ecological survey sites, you’ve likely encountered this problem with a histogram having a spike at zero that your model simply can’t account for. This post addresses that problem directly.

6.1 Why Standard Count Models Fall Short

Let’s start with the Poisson distribution as a reference point. The Poisson model assumes a single parameter λ, which simultaneously controls both the mean and the variance of the outcome. For many real-world count variables, this is already too restrictive — variance tends to exceed the mean (overdispersion), which is why the negative binomial, with its additional overdispersion parameter ϕ, is often preferred.

But even the negative binomial can struggle when zeros are structurally abundant. Consider the number of cigarettes smoked per day in a general population sample: many respondents are non-smokers who will never smoke regardless of covariates — their zero is a certainty, not a low-probability count. Others are occasional smokers who happen to report zero on the survey day. These two types of zeros come from qualitatively different processes, and conflating them leads to biased inference.

6.2 The Zero-Inflation Framework

The zero-inflated model is a finite mixture: with probability π, an observation is a structural zero (generated by a separate binary process); with probability 1 − π, it is drawn from a standard count distribution that can itself produce zeros. Formally, for a Zero-Inflated Poisson (ZIP):

\[ P(Y = 0) = \pi + (1 - \pi) e^{-\lambda} \] \[ P(Y = y) = (1 - \pi) \frac{e^{-\lambda} \lambda^y}{y!}, \quad y > 0 \]

The parameter π is the zero-inflation probability, often modeled using a logistic regression on its own set of covariates, while λ (or μ in the negative binomial variant) is modeled using a log link as before.

When overdispersion and excess zeros coexist — a common situation in ecological and health data — the Zero-Inflated Negative Binomial (ZINB) is typically preferred, combining the mixture structure with the NB’s variance flexibility.

6.3 Simulating Zero-Inflated Data in R

Before writing any Stan code, it’s useful to see what this phenomenon looks like in practice. We’ll simulate a dataset with two predictors and a meaningful proportion of structural zeros.

library(tidyverse)
library(rstan)
library(kableExtra)

set.seed(42)
N <- 5000

x1 <- rnorm(N)
x2 <- runif(N, -2, 2)

# Zero-inflation probability (structural zero process)
pi_zero <- plogis(-1 + 0.8 * x1)         # logistic model for pi

# Count process (negative binomial mean)
mu     <- exp(0.5 - 0.4 * x1 + 0.6 * x2) # log-linear model for mu
phi    <- 3                                # overdispersion

# Mix: structural zeros vs. NB draws
is_structural_zero <- rbinom(N, 1, pi_zero)
y_nb_raw           <- rnbinom(N, size = phi, mu = mu)
y_zinb             <- ifelse(is_structural_zero == 1, 0, y_nb_raw)

df_zinb <- data.frame(y = y_zinb, x1 = x1, x2 = x2)

cat("Proportion of zeros:", mean(y_zinb == 0), "\n")
cat("Expected from NB alone:", mean(dnbinom(0, size = phi, mu = mean(mu))), "\n")
## Proportion of zeros: 0.562
## Expected from NB alone: 0.271

The gap between the observed zero proportion (~56%) and what a negative binomial alone would predict (~27%) is precisely what zero-inflated models are designed to capture. A standard NB model fitted to this data would compensate by inflating the overdispersion estimate — absorbing the misfit into ϕ rather than correctly attributing it to the structural zero process.

A quick look at the distribution confirms the characteristic spike:

ggplot(df_zinb, aes(x = y)) +
  geom_histogram(binwidth = 1, fill = "#3B528B", color = "white", alpha = 0.85) +
  labs(title = "Simulated ZINB Data",
       subtitle = "Notice the excess mass at zero relative to a smooth NB curve",
       x = "Count (y)", y = "Frequency") +
  theme_minimal(base_size = 13)

6.4 Stan Model: Zero-Inflated Poisson

We begin with the simpler ZIP before graduating to ZINB. The Stan implementation uses the log_sum_exp trick for numerical stability when computing the mixture likelihood, which is standard practice in probabilistic programming.

data {
  int<lower=0> N;
  vector[N] x1;
  vector[N] x2;
  array[int] y[N];
}

parameters {
  // Count process (Poisson)
  real alpha_count;
  real beta1_count;
  real beta2_count;

  // Zero-inflation process (logistic)
  real alpha_zero;
  real beta1_zero;
}

model {
  // Priors
  alpha_count ~ normal(0, 5);
  beta1_count ~ normal(0, 2.5);
  beta2_count ~ normal(0, 2.5);
  alpha_zero  ~ normal(0, 5);
  beta1_zero  ~ normal(0, 2.5);

  for (n in 1:N) {
    real lambda  = exp(alpha_count + beta1_count * x1[n] + beta2_count * x2[n]);
    real logit_pi = alpha_zero + beta1_zero * x1[n];
    real log_pi   = log_inv_logit(logit_pi);
    real log1m_pi = log1m_inv_logit(logit_pi);

    if (y[n] == 0) {
      target += log_sum_exp(log_pi,
                            log1m_pi + poisson_lpmf(0 | lambda));
    } else {
      target += log1m_pi + poisson_lpmf(y[n] | lambda);
    }
  }
}

generated quantities {
  array[int] y_rep[N];
  for (n in 1:N) {
    real lambda   = exp(alpha_count + beta1_count * x1[n] + beta2_count * x2[n]);
    real pi_n     = inv_logit(alpha_zero + beta1_zero * x1[n]);
    int is_zero   = bernoulli_rng(pi_n);
    y_rep[n]      = is_zero ? 0 : poisson_rng(lambda);
  }
}

A few design choices worth noting:

  • Separate linear predictors for the count process and the zero-inflation process. It is entirely legitimate — and often theoretically motivated — to use different covariates in each component. For example, in a doctor visit model, age might drive visit frequency (count process) while health insurance status primarily predicts structural non-attendance (zero process).
  • log_sum_exp for the zero observation: this computes log(π + (1−π)·P(Y=0|λ)) in a numerically stable way, avoiding underflow when probabilities become very small.
  • generated quantities block for posterior predictive checks — essential for model criticism and something we should always include.

6.5 Stan Model: Zero-Inflated Negative Binomial

The ZINB is a natural extension. We simply replace the Poisson likelihood with the NB2 parameterization and add the overdispersion parameter ϕ.

data {
  int<lower=0> N;
  int<lower=0> K;
  matrix[N, K] X_count;    // predictors for count process
  matrix[N, K_zero] X_zero; // predictors for zero-inflation (can differ)
  int<lower=0> K_zero;
  array[int] y[N];
}

parameters {
  vector[K]      beta_count;  // count process coefficients
  vector[K_zero] beta_zero;   // zero-inflation coefficients
  real<lower=0>  phi;         // overdispersion
}

model {
  // Priors — Gelman (2008) recommendations
  beta_count[1] ~ cauchy(0, 10);
  for (k in 2:K)      beta_count[k] ~ cauchy(0, 2.5);
  beta_zero[1]  ~ cauchy(0, 10);
  for (k in 2:K_zero) beta_zero[k]  ~ cauchy(0, 2.5);
  phi ~ gamma(2, 0.1); // weakly informative; keeps phi positive and not too large

  for (n in 1:N) {
    real mu       = exp(dot_product(X_count[n], beta_count));
    real logit_pi = dot_product(X_zero[n], beta_zero);
    real log_pi   = log_inv_logit(logit_pi);
    real log1m_pi = log1m_inv_logit(logit_pi);

    if (y[n] == 0) {
      target += log_sum_exp(log_pi,
                            log1m_pi + neg_binomial_2_lpmf(0 | mu, phi));
    } else {
      target += log1m_pi + neg_binomial_2_lpmf(y[n] | mu, phi);
    }
  }
}

One subtlety here is the prior on ϕ. A gamma(2, 0.1) prior places most of its mass in the range , which implies moderate-to-mild overdispersion — a reasonable weakly informative choice for most empirical count data. If your domain knowledge suggests more severe overdispersion (e.g., rare event counts), you can shift toward gamma(1, 0.5).

6.6 Fitting the ZINB Model in R

stan_zinb_mod <- "
data {
  int<lower=0> N;
  vector[N] x1;
  vector[N] x2;
  int<lower=0> y[N];
}
parameters {
  real alpha_count;
  real beta1_count;
  real beta2_count;
  real alpha_zero;
  real beta1_zero;
  real<lower=0> phi;
}
model {
  alpha_count ~ normal(0, 5);
  beta1_count ~ cauchy(0, 2.5);
  beta2_count ~ cauchy(0, 2.5);
  alpha_zero  ~ normal(0, 5);
  beta1_zero  ~ cauchy(0, 2.5);
  phi ~ gamma(2, 0.1);

  for (n in 1:N) {
    real mu       = exp(alpha_count + beta1_count * x1[n] + beta2_count * x2[n]);
    real logit_pi = alpha_zero + beta1_zero * x1[n];
    real log_pi   = log_inv_logit(logit_pi);
    real log1m_pi = log1m_inv_logit(logit_pi);

    if (y[n] == 0) {
      target += log_sum_exp(log_pi,
                            log1m_pi + neg_binomial_2_lpmf(0 | mu, phi));
    } else {
      target += log1m_pi + neg_binomial_2_lpmf(y[n] | mu, phi);
    }
  }
}
"

writeLines(stan_zinb_mod, con = "stan_zinb.stan")

stan_data_zinb <- list(
  N  = N,
  x1 = df_zinb$x1,
  x2 = df_zinb$x2,
  y  = df_zinb$y
)

fit_zinb <- rstan::stan(
  file    = "stan_zinb.stan",
  data    = stan_data_zinb,
  chains  = 4,
  iter    = 2000,
  warmup  = 1000,
  control = list(adapt_delta = 0.95)  # recommended for mixture models
)

Note the adapt_delta = 0.95 setting. Zero-inflated models tend to have somewhat more complex posterior geometries — particularly when π and λ are weakly identified from the same observations — and giving the NUTS sampler a smaller step size helps avoid divergent transitions.

The true generative values were alpha_count = 0.5, beta1_count = -0.4, beta2_count = 0.6, alpha_zero = -1, beta1_zero = 0.8, and phi = 3. The posterior means are well-calibrated, and Rhat ≈ 1 for all parameters confirms convergence across chains.

Interpreting the two sets of coefficients requires care:

  • Count process coefficients (beta_count) are on the log scale for μ: beta2_count ≈ 0.60 means a one-unit increase in x2 is associated with a multiplicative change of e⁰·⁶ ≈ 1.82 in the expected count, among those not in the structural zero group.
  • Zero-inflation coefficients (beta_zero) are on the log-odds scale for π: beta1_zero ≈ 0.81 means a one-unit increase in x1 increases the log-odds of being a structural zero by 0.81, corresponding to an odds ratio of e⁰·⁸¹ ≈ 2.25.

6.8 Model Comparison: ZIP vs. ZINB vs. NB

A natural question is whether the extra complexity of the zero-inflated model is justified. The Bayesian answer is to compute the Expected Log Predictive Density (ELPD) via Leave-One-Out cross-validation, available through the loo package:

library(loo)

log_lik_zinb <- extract_log_lik(fit_zinb, merge_chains = FALSE)
loo_zinb     <- loo(log_lik_zinb, r_eff = relative_eff(exp(log_lik_zinb)))

# Compare against a plain NB fit (fit_rstan3 from the previous post)
log_lik_nb   <- extract_log_lik(fit_nb, merge_chains = FALSE)
loo_nb       <- loo(log_lik_nb, r_eff = relative_eff(exp(log_lik_nb)))

loo_compare(loo_zinb, loo_nb)

A positive ELPD difference favoring the ZINB model is strong evidence that the zero-inflation component is capturing real structure in the data — not just overfitting. When the difference is small relative to its standard error, you might reasonably prefer the simpler NB for parsimony.

6.9 Practical Guidance and Caveats

A few lessons from practice worth internalizing:

  • Identifiability is the main hazard. If the count model predicts very low μ, the NB component already produces many zeros, and the data cannot tell the two components apart. This manifests as a funnel-shaped posterior when you plot π against μ — a sign that your model may be over-parameterized or your covariates are insufficient.
  • Posterior predictive checks are non-negotiable. Always plot the posterior predictive distribution of the proportion of zeros (i.e., mean(y_rep == 0) across posterior draws) against the observed proportion. If your ZINB still underestimates zeros, you may have a more severe structural issue with the data.
  • Consider hurdle models as an alternative. Where zero-inflated models treat zero as a mixture of two processes, hurdle models have a hard boundary: a binary model decides whether the count is zero, and a truncated count model handles positive values. Hurdle models are sometimes more interpretable when structural and sampling zeros genuinely cannot co-occur.
  • Rhat and n_eff are your friends. Mixture models can be slow to mix. If n_eff for π-related parameters is below ~400, increase iter or simplify the zero-inflation component.

6.10 Conclusion

Zero-inflated models occupy an important niche in the practitioner’s toolkit. They extend naturally from the negative binomial framework we developed in the last post — the ZINB is, in a sense, the negative binomial plus an honest acknowledgment that some zeros are structurally different from others. The Bayesian implementation in Stan gives us full uncertainty quantification over both the count process and the zero-inflation probability, and the generated quantities block makes posterior predictive checking straightforward.

Future posts will extend these ideas to hurdle models and to settings where the zero-inflation probability itself varies as a smooth function of covariates — effectively a semiparametric treatment of the structural zero process.
