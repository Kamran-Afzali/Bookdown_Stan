Chapter 8 Bayesian Regression Models for Categorical Outcomes

Our previous posts walked through Bayesian regression for binary and count outcomes in R using Stan. This post continues in that spirit, covering two models for categorical responses: ordinal regression for outcomes with a natural ordering, and categorical (multinomial) regression for unordered classes.

The Bayesian workflow remains the same throughout: we specify a generative model, place prior distributions over the unknown parameters, and use Markov Chain Monte Carlo (MCMC) to draw samples from the posterior. What changes is the likelihood — and that choice has real consequences for how the model treats the structure of the outcome.

8.1 Ordinal Regression

Consider a response variable \(y_i \in \{1, 2, \ldots, K\}\) where the categories carry a meaningful order — a Likert scale being the canonical example. Standard logistic regression discards this ordering entirely by collapsing to a binary contrast, and ordinary linear regression imposes cardinal spacing that the data rarely support. Ordinal regression occupies the natural middle ground.

The most widely used formulation is the cumulative logit model, sometimes called the proportional odds model. The key idea is to model the cumulative probabilities \(P(y_i \leq k)\) rather than the individual class probabilities. Concretely, we write

\[ \log \frac{P(y_i \leq k \mid \mathbf{x}_i)}{P(y_i > k \mid \mathbf{x}_i)} = c_k - \mathbf{x}_i^\top \boldsymbol{\beta}, \quad k = 1, \ldots, K-1, \]

where \(c_1 < c_2 < \cdots < c_{K-1}\) are ordered cutpoints on the latent scale and \(\boldsymbol{\beta}\) is the vector of predictor effects. The proportional odds assumption is embedded in this formulation: the same \(\boldsymbol{\beta}\) applies at every threshold, so the predictor shifts the entire cumulative distribution uniformly rather than affecting each boundary differently.

An equivalent and more intuitive derivation invokes a latent variable \(z_i\). We imagine an unobserved continuous score

\[ z_i = \mathbf{x}_i^\top \boldsymbol{\beta} + \varepsilon_i, \quad \varepsilon_i \sim \text{Logistic}(0, 1), \]

and the observed category is simply determined by where ( z_i $ lands relative to the cutpoints:

\[ y_i = k \iff c_{k-1} < z_i \leq c_k, \]

with \(c_0 = -\infty\) and ( c_K = +$ by convention. This latent-variable view is exactly how we simulate the data below.

8.1.1 Simulating Ordinal Data

library(tidyverse)
library(kableExtra)
library(rstan)

set.seed(1234)

N <- 5000
x1 <- rnorm(N)

eta        <- 1.2 * x1
cutpoints  <- c(-1.5, -0.2, 1.0, 2.0)

z <- eta + rlogis(N)

y <- case_when(
  z <= cutpoints[1] ~ 1,
  z <= cutpoints[2] ~ 2,
  z <= cutpoints[3] ~ 3,
  z <= cutpoints[4] ~ 4,
  TRUE              ~ 5
)

df_ord <- data.frame(y = y, x1 = x1)
head(df_ord) %>% kableExtra::kable()

We generated \(N = 5{,}000\) observations with a single continuous predictor and a true coefficient of \(\beta = 1.2\). The four cutpoints partition the latent axis into five observed categories.

8.1.2 Stan Model — Single Predictor

model_ord <- "
data {
  int<lower=1> N;
  int<lower=2> K_cat;
  vector[N] x;
  array[N] int<lower=1,upper=K_cat> y;
}
parameters {
  real beta;
  ordered[K_cat - 1] c;
}
model {
  beta ~ normal(0, 2.5);
  c    ~ normal(0, 5);

  y ~ ordered_logistic(beta * x, c);
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N)
    y_rep[n] = ordered_logistic_rng(beta * x[n], c);
}
"
writeLines(model_ord, con = "model_ord.stan")

Stan’s ordered[K_cat - 1] type enforces the constraint \(c_1 < c_2 < \cdots < c_{K-1}\) directly in the parameterization, so we do not need to worry about imposing it manually. The prior \(\beta \sim \mathcal{N}(0, 2.5)\) is weakly informative on the log-odds scale, while \(c_k \sim \mathcal{N}(0, 5)\) allows the cutpoints to spread reasonably across the latent axis.

stan_data_ord <- list(
  N     = nrow(df_ord),
  K_cat = 5,
  x     = df_ord$x1,
  y     = df_ord$y
)

fit_ord <- rstan::stan(
  file   = "model_ord.stan",
  data   = stan_data_ord,
  chains = 4,
  iter   = 2000,
  warmup = 1000,
  seed   = 1234
)

fit_ord

The posterior mean for \(\beta\) should sit close to 1.2, and the recovered cutpoints should approximate \((-1.5,\ -0.2,\ 1.0,\ 2.0)\). A positive coefficient tells us that higher values of \(x_1\) push probability mass toward higher outcome categories — exactly as the latent-variable interpretation suggests.

8.1.3 Extending to Multiple Predictors

The model generalises straightforwardly. With a predictor matrix \(\mathbf{X} \in \mathbb{R}^{N \times K}\), the linear predictor becomes \(\boldsymbol{\eta} = \mathbf{X}\boldsymbol{\beta}\), and the rest of the model is unchanged.

set.seed(1234)

x1 <- rnorm(N); x2 <- rnorm(N); x3 <- rnorm(N)

eta       <- 1 + 1.5 * x1 - 0.8 * x2 + 0.6 * x3
cutpoints <- c(-2, -0.5, 0.8, 2.2)

z <- eta + rlogis(N)
y <- case_when(
  z <= cutpoints[1] ~ 1,
  z <= cutpoints[2] ~ 2,
  z <= cutpoints[3] ~ 3,
  z <= cutpoints[4] ~ 4,
  TRUE              ~ 5
)

df_ord2 <- data.frame(y = y, x1 = x1, x2 = x2, x3 = x3)
head(df_ord2) %>% kableExtra::kable()
model_ord2 <- "
data {
  int<lower=1> N;
  int<lower=1> K;
  int<lower=2> K_cat;
  matrix[N, K] X;
  array[N] int<lower=1,upper=K_cat> y;
}
parameters {
  vector[K] beta;
  ordered[K_cat - 1] c;
}
model {
  beta ~ normal(0, 2.5);
  c    ~ normal(0, 5);

  y ~ ordered_logistic(X * beta, c);
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N)
    y_rep[n] = ordered_logistic_rng(X[n] * beta, c);
}
"
writeLines(model_ord2, con = "model_ord2.stan")
X_ord <- as.matrix(df_ord2[, c("x1", "x2", "x3")])

stan_data_ord2 <- list(
  N     = nrow(df_ord2),
  K     = ncol(X_ord),
  K_cat = 5,
  X     = X_ord,
  y     = df_ord2$y
)

fit_ord2 <- rstan::stan(
  file   = "model_ord2.stan",
  data   = stan_data_ord2,
  chains = 4,
  iter   = 2000,
  warmup = 1000,
  seed   = 1234
)

fit_ord2

The posterior summaries should recover \(\boldsymbol{\beta} \approx (1.5,\ -0.8,\ 0.6)\). It is also worth running posterior predictive checks to confirm that the model reproduces the observed category frequencies — a model that fits the coefficients well but misses the marginal distribution is still a flawed model.

8.2 Categorical / Multinomial Regression

When the outcome has more than two unordered categories — preferred transport mode, product choice, clinical diagnosis type — the proportional odds structure is no longer meaningful. Here we use categorical regression, also called multinomial logistic regression.

The model works by assigning each observation a vector of log-odds \(\boldsymbol{\eta}_i \in \mathbb{R}^K\) and applying the softmax transformation to obtain class probabilities:

\[ P(y_i = k \mid \mathbf{x}_i) = \frac{\exp(\eta_{ik})}{\sum_{j=1}^{K} \exp(\eta_{ij})}, \quad k = 1, \ldots, K. \]

Because the probabilities sum to one, the model is not identified without a constraint. The standard fix is to pin one class — say class 1 — as the reference by setting \(\eta_{i1} = 0\), so that

\[ \eta_{ik} = \alpha_k + \mathbf{x}_i^\top \boldsymbol{\beta}_k, \quad k = 2, \ldots, K. \]

Every coefficient \(\beta_{kj}\) is then interpreted as the change in the log-odds of class \(k\) relative to the baseline class, per unit increase in predictor \(j\). This is a direct generalisation of binary logistic regression: with \(K = 2\), the two formulations are identical.

8.2.1 Simulating Multinomial Data

set.seed(1234)

N  <- 5000
x1 <- rnorm(N)

eta1 <- 0.5 + 1.0 * x1
eta2 <- -0.5 - 1.2 * x1

exp_eta1 <- exp(eta1)
exp_eta2 <- exp(eta2)
denom    <- 1 + exp_eta1 + exp_eta2

p1 <- 1        / denom
p2 <- exp_eta1 / denom
p3 <- exp_eta2 / denom

y <- map_int(1:N, function(i) sample(1:3, size = 1, prob = c(p1[i], p2[i], p3[i])))

df_cat <- data.frame(y = y, x1 = x1)
head(df_cat) %>% kableExtra::kable()

The denominator \(1 + e^{\eta_1} + e^{\eta_2}\) is precisely the softmax normaliser with class 1 fixed as the reference. This ensures \(p_1 + p_2 + p_3 = 1\) for every observation.

8.2.2 Stan Model — Single Predictor

model_cat <- "
data {
  int<lower=1> N;
  int<lower=2> K_cat;
  vector[N] x;
  array[N] int<lower=1,upper=K_cat> y;
}
parameters {
  vector[K_cat - 1] alpha;
  vector[K_cat - 1] beta;
}
model {
  alpha ~ normal(0, 5);
  beta  ~ normal(0, 2.5);

  for (n in 1:N) {
    vector[K_cat] eta;
    eta[1] = 0;
    for (k in 2:K_cat)
      eta[k] = alpha[k - 1] + beta[k - 1] * x[n];
    y[n] ~ categorical_logit(eta);
  }
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N) {
    vector[K_cat] eta;
    eta[1] = 0;
    for (k in 2:K_cat)
      eta[k] = alpha[k - 1] + beta[k - 1] * x[n];
    y_rep[n] = categorical_logit_rng(eta);
  }
}
"
writeLines(model_cat, con = "model_cat.stan")
stan_data_cat <- list(
  N     = nrow(df_cat),
  K_cat = 3,
  x     = df_cat$x1,
  y     = df_cat$y
)

fit_cat <- rstan::stan(
  file   = "model_cat.stan",
  data   = stan_data_cat,
  chains = 4,
  iter   = 2000,
  warmup = 1000,
  seed   = 1234
)

fit_cat

The categorical_logit distribution in Stan handles the softmax internally, so we only need to pass the unnormalised log-odds vector \(\boldsymbol{\eta}\). The posterior for \(\alpha\) and \(\beta\) should recover the true values \((\alpha_2, \alpha_3) = (0.5, -0.5)\) and \((\beta_2, \beta_3) = (1.0, -1.2)\).

8.2.3 Extending to Multiple Predictors

With a predictor matrix \(\mathbf{X} \in \mathbb{R}^{N \times K}\), each non-reference class gets its own coefficient vector, so the parameter block becomes a matrix \(\mathbf{B} \in \mathbb{R}^{(K_{\text{cat}}-1) \times K}\).

set.seed(1234)

x1 <- rnorm(N); x2 <- rnorm(N); x3 <- rnorm(N)

eta1 <-  0.5 + 1.2 * x1 - 0.7 * x2 + 0.3 * x3
eta2 <- -0.8 - 1.0 * x1 + 0.9 * x2 - 0.4 * x3

exp_eta1 <- exp(eta1); exp_eta2 <- exp(eta2)
denom    <- 1 + exp_eta1 + exp_eta2

p1 <- 1 / denom; p2 <- exp_eta1 / denom; p3 <- exp_eta2 / denom

y <- map_int(1:N, function(i) sample(1:3, size = 1, prob = c(p1[i], p2[i], p3[i])))

df_cat2 <- data.frame(y = y, x1 = x1, x2 = x2, x3 = x3)
head(df_cat2) %>% kableExtra::kable()
model_cat2 <- "
data {
  int<lower=1> N;
  int<lower=1> K;
  int<lower=2> K_cat;
  matrix[N, K] X;
  array[N] int<lower=1,upper=K_cat> y;
}
parameters {
  matrix[K_cat - 1, K] beta;
  vector[K_cat - 1] alpha;
}
model {
  to_vector(beta) ~ normal(0, 2.5);
  alpha           ~ normal(0, 5);

  for (n in 1:N) {
    vector[K_cat] eta;
    eta[1] = 0;
    for (k in 2:K_cat)
      eta[k] = alpha[k - 1] + X[n] * beta[k - 1]';
    y[n] ~ categorical_logit(eta);
  }
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N) {
    vector[K_cat] eta;
    eta[1] = 0;
    for (k in 2:K_cat)
      eta[k] = alpha[k - 1] + X[n] * beta[k - 1]';
    y_rep[n] = categorical_logit_rng(eta);
  }
}
"
writeLines(model_cat2, con = "model_cat2.stan")
X_cat <- as.matrix(df_cat2[, c("x1", "x2", "x3")])

stan_data_cat2 <- list(
  N     = nrow(df_cat2),
  K     = ncol(X_cat),
  K_cat = 3,
  X     = X_cat,
  y     = df_cat2$y
)

fit_cat2 <- rstan::stan(
  file   = "model_cat2.stan",
  data   = stan_data_cat2,
  chains = 4,
  iter   = 2000,
  warmup = 1000,
  seed   = 1234
)

fit_cat2

With enough data, the posterior should recover the six true slope values. In practice, raw coefficient tables become hard to interpret when there are several predictors and classes — plotting the predicted class probabilities \(\hat{P}(y = k \mid x)\) as a function of each predictor while marginalising the others is often far more informative.

8.3 Model Checking

Fitting the model is only the first step. For both model types, it is essential to verify that \(\hat{R} \approx 1\) and that the effective sample sizes are adequate before trusting posterior summaries. Stan will warn you when chains have not mixed, but it is good practice to inspect trace plots as well.

Posterior predictive checks offer a complementary perspective. By comparing the observed category frequencies to the distribution of frequencies generated under \(y^{\text{rep}}\), we can detect systematic misfit that convergence diagnostics alone would miss. For the ordinal model, this check also gives informal evidence about the proportional odds assumption: if the model consistently underestimates the probability of one or two categories, the constant-\(\boldsymbol{\beta}\) constraint may be too restrictive and a partial proportional odds model or a fully non-proportional formulation could be worth exploring.

For categorical regression the analogous check is straightforward — compare observed and predicted class proportions, possibly stratified by a covariate to see whether the softmax structure holds across the range of \(\mathbf{X}\).
