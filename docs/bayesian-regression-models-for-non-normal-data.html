Chapter 5 Bayesian Regression Models for Non-Normal Data

Our last post covered how to do Bayesian regression for normally distributed data in R using STAN. In this post, we’ll look at how to fit regression models for non-normal data in STAN using two common distributions seen in empirical data, namely the binomial and the negative binomial. In Bayesian modeling, we define a statistical model, choose prior distributions for the unknown parameters, and use sampling methods such as Markov Chain Monte Carlo (MCMC) to estimate the posterior distribution. github

5.1 Logistic Regression

The likelihood of a binary outcome, such as pass or fail, is commonly modeled using logistic regression. This is done by linking the probability of success to a linear predictor through the logit function, which keeps the fitted probabilities between 0 and 1. In practice, the coefficients describe changes in log-odds, and exponentiating them gives odds ratios. mc-stan

The following code simulates data for a logistic regression model with one predictor and an intercept.

library(tidyverse)
library(kableExtra)
library(rstan)

set.seed(1234)

x1 <- rnorm(10000)
z  <- 1 + 2 * x1
pr <- 1 / (1 + exp(-z))
y  <- rbinom(10000, 1, pr)

df <- data.frame(y = y, x1 = x1)

head(df) %>% kableExtra::kable()

As a quick check, we can fit the same model with glm() using a binomial family. Since the data were simulated from a logistic model with intercept 1 and slope 2, the estimates should be close to those values. mc-stan

glm(y ~ x1, data = df, family = "binomial") %>%
  summary() %>%
  pluck(coefficients) %>%
  kableExtra::kable()

Now let’s fit the Bayesian version in STAN. Here we add weakly informative priors to stabilize estimation and make the model more appropriate as a default teaching example. Weakly informative priors are commonly recommended for logistic-type models, especially when predictors are reasonably scaled. projecteuclid

model_stan <- "
data {
  int<lower=0> N;
  vector[N] x;
  array[N] int<lower=0,upper=1> y;
}
parameters {
  real alpha;
  real beta;
}
model {
  alpha ~ normal(0, 5);
  beta  ~ normal(0, 2.5);

  y ~ bernoulli_logit(alpha + beta * x);
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N) {
    y_rep[n] = bernoulli_logit_rng(alpha + beta * x[n]);
  }
}
"
writeLines(model_stan, con = "model_stan.stan")
cat(model_stan)
stan_data <- list(
  N = nrow(df),
  x = df$x1,
  y = df$y
)

fit_rstan <- rstan::stan(
  file = "model_stan.stan",
  data = stan_data,
  chains = 4,
  iter = 2000,
  warmup = 1000,
  seed = 1234
)
fit_rstan

The posterior means should be close to the true values used to generate the data. When checking the fit, it is also good practice to look at convergence diagnostics such as Rhat, effective sample size, and posterior predictive checks based on y_rep. Posterior predictive checks help assess whether the model can reproduce data that look like the observed sample. mc-stan

This can also be expanded to several predictors. Below is the code for a logistic regression with three predictors and an intercept.

set.seed(1234)

x1 <- rnorm(10000)
x2 <- rnorm(10000)
x3 <- rnorm(10000)

z  <- 1 + 2 * x1 + 3 * x2 - 1 * x3
pr <- 1 / (1 + exp(-z))
y  <- rbinom(10000, 1, pr)

df2 <- data.frame(y = y, x1 = x1, x2 = x2, x3 = x3)

head(df2) %>% kableExtra::kable()
glm(y ~ x1 + x2 + x3, data = df2, family = "binomial") %>%
  summary() %>%
  pluck(coefficients) %>%
  kableExtra::kable()

The STAN model now uses a predictor matrix and a coefficient vector.

model_stan2 <- "
data {
  int<lower=0> N;
  int<lower=1> K;
  matrix[N, K] X;
  array[N] int<lower=0,upper=1> y;
}
parameters {
  real alpha;
  vector[K] beta;
}
model {
  alpha ~ normal(0, 5);
  beta  ~ normal(0, 2.5);

  y ~ bernoulli_logit(alpha + X * beta);
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N) {
    y_rep[n] = bernoulli_logit_rng(alpha + X[n] * beta);
  }
}
"
writeLines(model_stan2, con = "model_stan2.stan")
cat(model_stan2)
predictors <- as.matrix(df2[, c("x1", "x2", "x3")])

stan_data2 <- list(
  N = nrow(df2),
  K = ncol(predictors),
  X = predictors,
  y = df2$y
)

fit_rstan2 <- rstan::stan(
  file = "model_stan2.stan",
  data = stan_data2,
  chains = 4,
  iter = 2000,
  warmup = 1000,
  seed = 1234
)
fit_rstan2

Again, the posterior means should be close to the true simulated coefficients. In applied work, it is worth centering or scaling predictors before fitting the model, because prior interpretation is much clearer when coefficients operate on comparable scales. This is especially important when using weakly informative priors. github

5.2 Negative Binomial

The Poisson distribution is a popular choice for modeling count data, but it assumes that the variance is equal to the mean. When the variance exceeds the mean, the data are said to be overdispersed, and the negative binomial distribution is often a better choice. In STAN, the neg_binomial_2(mu, phi) parameterization uses mu as the mean and phi as a positive overdispersion parameter; as phi becomes large, the model approaches a Poisson distribution. mc-stan

Let’s simulate some data from a negative binomial regression model and fit a STAN model to them.

library(MASS)

set.seed(1234)

N <- 10000
df3 <- data.frame(
  x1 = runif(N, -2, 2),
  x2 = runif(N, -2, 2)
)

X <- model.matrix(~ x1 * x2, df3)
K <- ncol(X)

betas <- runif(K, -1, 1)
phi   <- 5

mu   <- exp(X %*% betas)
y_nb <- rnbinom(N, size = phi, mu = mu)

hist(y_nb)

As a frequentist comparison, we can fit a negative binomial model using glm.nb().

MASS::glm.nb(y_nb ~ x1 * x2, data = df3) %>%
  summary() %>%
  pluck(coefficients) %>%
  kableExtra::kable()

Now we fit the Bayesian version in STAN. One important correction here is that the dispersion parameter must be constrained to be positive. Since phi in neg_binomial_2(mu, phi) is defined on the positive real line, it should be declared with a lower bound in the parameters block. mc-stan

stan_mod <- "
data {
  int<lower=1> N;
  int<lower=1> K;
  array[N] int<lower=0> y;
  matrix[N, K] X;
}
parameters {
  vector[K] beta;
  real<lower=0> phi;
}
transformed parameters {
  vector[N] mu;
  mu = exp(X * beta);
}
model {
  beta [aimspress](https://www.aimspress.com/article/doi/10.3934/math.2024883) ~ normal(0, 5);
  beta[2:K] ~ normal(0, 2.5);
  phi ~ exponential(1);

  y ~ neg_binomial_2(mu, phi);
}
generated quantities {
  array[N] int y_rep;
  for (n in 1:N) {
    y_rep[n] = neg_binomial_2_rng(mu[n], phi);
  }
}
"
writeLines(stan_mod, con = "stan_mod.stan")
cat(stan_mod)
stan_data3 <- list(
  N = N,
  K = K,
  X = X,
  y = y_nb
)
fit_rstan3 <- rstan::stan(
  file = "stan_mod.stan",
  data = stan_data3,
  chains = 4,
  iter = 2000,
  warmup = 1000,
  seed = 1234
)
fit_rstan3 %>% summary()

The posterior means should recover the simulated regression coefficients and the dispersion parameter reasonably well. As before, it is useful to examine convergence diagnostics and posterior predictive checks, since a good Bayesian workflow does not stop at parameter estimation alone. Prior predictive and posterior predictive checks are both recommended for assessing whether the priors and likelihood imply realistic data patterns. mc-stan

5.3 Conclusion

This tutorial provided a quick overview of how to fit logistic and negative binomial regression models with the Bayesian software STAN using the rstan interface. The key ideas are to choose a likelihood appropriate for the outcome, use sensible weakly informative priors, constrain parameters to their valid domains, and check model fit using predictive diagnostics rather than relying only on coefficient summaries. mc-stan
